## 🎯 什么是音视频剪辑？

### 简单理解

**音视频剪辑**就是从一个视频中"截取"一部分，或者把多个视频"拼接"在一起。

### 生活中的比喻

想象你用手机拍了一段视频：
- 🎬 **原视频**：10 秒（00:00 - 00:10）
- ✂️ **剪辑操作**：只保留 3-7 秒
- 📹 **输出结果**：5 秒（00:03 - 00:07）

### 常见的剪辑操作

| 操作 | 说明 | 例子 |
|-----|------|------|
| **裁剪** | 截取视频的一部分 | 只保留 3-7 秒 |
| **拼接** | 把多个视频连在一起 | 视频 A + 视频 B |
| **分割** | 把一个视频切成两段 | 在 5 秒处切开 |
| **变速** | 改变播放速度 | 2x 快进 |
| **混音** | 替换或混合音频 | 背景音乐替换 |

## 🎯 剪辑的核心原理

### 关键概念：时间戳（Timestamp）

每个视频帧和音频帧都有一个**时间戳**，表示什么时候应该播放。

```
原始视频时间线：
|-------10秒视频-------|
0s    1s    2s    3s    4s    5s    6s    7s    8s    9s    10s
|-----|-----|-----|-----|-----|-----|-----|-----|-----|-----|
       [===========]
        要保留的部分

如果我们想裁剪 3-7 秒的部分：
- 需要保留：3s、4s、5s、6s、7s 的帧
- 需要删除：0s、1s、2s、8s、9s 的帧
```

### 关键概念：关键帧（I-Frame）

**重要**：视频剪辑必须从关键帧开始！

```
视频帧序列：
I(0s) → P(0.03s) → P(0.06s) → B(0.09s) → I(0.12s) → P(0.15s) ...
 ↑                                    ↑
 关键帧                               关键帧

如果我们想从 0.09s 开始裁剪：
- ❌ 不能从 B(0.09s) 开始（它依赖 P(0.06s)）
- ✅ 必须从最近的 I(0.12s) 开始
```

### 剪辑的基本步骤

```
1️⃣ 解析原视频
   └── 了解视频有多少帧、每个帧的时间戳

2️⃣ 找到裁剪点
   └── 确定从哪里开始、到哪里结束
   └── 调整到关键帧位置

3️⃣ 提取需要的帧
   └── 只复制需要的视频帧和音频帧

4️⃣ 创建新文件
   └── 用 MediaMuxer 把提取的帧写入新文件

5️⃣ 更新元数据
   └── 调整时间戳、时长等信息
```

## 🎯 方案一：直接裁剪

### 原理

直接复制原文件的指定部分，写入新文件。

### 适用情况

- ✅ 剪辑点刚好在关键帧位置
- ✅ 对精度要求不高
- ✅ 想要快速完成

### 实现步骤

```kotlin
/**
 * 简单裁剪方案
 * 直接复制指定时间范围的视频数据
 */
fun simpleTrim(
    inputPath: String,
    outputPath: String,
    startTimeMs: Long,
    endTimeMs: Long
) {
    val extractor = MediaExtractor()
    extractor.setDataSource(inputPath)

    val muxer = MediaMuxer(outputPath, MediaMuxer.OutputFormat.MUXER_OUTPUT_MPEG_4)

    // 1. 遍历所有轨道
    for (i in 0 until extractor.trackCount) {
        val format = extractor.getTrackFormat(i)
        val mime = format.getString(MediaFormat.KEY_MIME) ?: continue

        // 2. 只处理视频和音频轨道
        if (mime.startsWith("video/") || mime.startsWith("audio/")) {
            extractor.selectTrack(i)
            val trackIndex = muxer.addTrack(format)

            muxer.start()

            // 3. 跳到开始时间
            extractor.seekTo(startTimeMs * 1000, MediaExtractor.SEEK_TO_CLOSEST_SYNC)

            // 4. 循环读取并写入数据
            val buffer = ByteBuffer.allocate(1024 * 1024)
            val bufferInfo = MediaCodec.BufferInfo()

            while (true) {
                val sampleSize = extractor.readSampleData(buffer, 0)

                if (sampleSize < 0 || extractor.sampleTime > endTimeMs * 1000) {
                    break
                }

                bufferInfo.offset = 0
                bufferInfo.size = sampleSize
                bufferInfo.presentationTimeUs = extractor.sampleTime - startTimeMs * 1000
                bufferInfo.flags = extractor.sampleFlags

                muxer.writeSampleData(trackIndex, buffer, bufferInfo)
                extractor.advance()
            }
        }
    }

    muxer.stop()
    muxer.release()
    extractor.release()
}
```

### 问题分析

```
❌ 问题：时间戳不连续
原因：直接裁剪可能导致时间戳跳跃

原始：
帧1 (0s) → 帧2 (0.03s) → 帧3 (0.06ms) → ...
        ↓ 裁剪掉
输出：
帧1 (0s) → 帧3 (0.06s)  ← 时间戳跳跃！
```

## 🎯 方案二：重新编码裁剪

### 原理

完全解码视频，然后只编码需要的部分。

### 适用情况

- ✅ 需要精确剪辑
- ✅ 需要保持高质量
- ✅ 不在乎处理时间

### 实现步骤

```kotlin
/**
 * 完整解码-重新编码方案
 * 1. 解码所有帧
 * 2. 只编码需要的帧
 * 3. 混合输出
 */
fun preciseTrim(
    inputPath: String,
    outputPath: String,
    startTimeMs: Long,
    endTimeMs: Long
) {
    // ========== 第一阶段：解码 ==========

    val extractor = MediaExtractor()
    extractor.setDataSource(inputPath)

    // 找到视频轨道并解码
    val decoder = createDecoder(extractor, "video/")
    val videoFrames = mutableListOf<FrameData>()

    // 解码所有视频帧
    while (true) {
        val frame = decoder.decodeFrame(extractor)
        if (frame == null) break

        // 只保留时间范围内的帧
        if (frame.timestamp in startTimeMs..endTimeMs) {
            videoFrames.add(frame)
        }
    }

    // ========== 第二阶段：编码 ==========

    val encoder = createEncoder(videoFormat)
    val encodedData = mutableListOf<ByteBuffer>()

    // 重新编码（时间戳从0开始）
    for ((index, frame) in videoFrames.withIndex()) {
        val newTimestamp = (index * (1000000 / 30)).toLong() // 假设30fps
        val encoded = encoder.encodeFrame(frame.data, newTimestamp)
        encodedData.add(encoded)
    }

    // ========== 第三阶段：混合输出 ==========

    val muxer = MediaMuxer(outputPath, MediaMuxer.OutputFormat.MUXER_OUTPUT_MPEG_4)
    val videoTrack = muxer.addTrack(videoFormat)
    muxer.start()

    for (data in encodedData) {
        muxer.writeSampleData(videoTrack, data, bufferInfo)
    }

    muxer.stop()
    muxer.release()

    // 释放资源...
}
```

### 问题分析

```
⚠️ 缺点：
1. 速度慢 - 需要解码和重新编码整个视频
2. 质量损失 - 每次编码都会降低质量
3. 资源消耗 - CPU 和内存占用大
```

## 🎯 方案三：使用 MediaMuxer 精确剪辑

### 原理

利用 MediaMuxer 的特性，精确控制时间戳和轨道。

### 适用情况

- ✅ 大多数剪辑场景
- ✅ 需要在关键帧位置剪辑
- ✅ 保持原视频质量

### 实现步骤

```kotlin
/**
 * MediaMuxer 精确剪辑方案
 * 结合方案一和方案二的优点
 */
class VideoClipper(
    private val inputPath: String,
    private val outputPath: String
) {
    private var extractor: MediaExtractor? = null
    private var muxer: MediaMuxer? = null

    /**
     * 执行剪辑
     * @param startTimeMs 开始时间（毫秒）
     * @param endTimeMs 结束时间（毫秒）
     */
    fun clip(startTimeMs: Long, endTimeMs: Long) {
        try {
            // ========== 准备阶段 ==========

            // 1. 创建 Extractor
            extractor = MediaExtractor()
            extractor!!.setDataSource(inputPath)

            // 2. 创建 Muxer
            muxer = MediaMuxer(outputPath, MediaMuxer.OutputFormat.MUXER_OUTPUT_MPEG_4)

            // 3. 找到视频和音频轨道
            val trackIndices = mutableMapOf<Int, Int>() // 原轨道 -> 新轨道

            for (i in 0 until extractor!!.trackCount) {
                val format = extractor!!.getTrackFormat(i)
                val mime = format.getString(MediaFormat.KEY_MIME) ?: continue

                // 只处理视频和音频
                if (mime.startsWith("video/") || mime.startsWith("audio/")) {
                    extractor!!.selectTrack(i)
                    val newTrackIndex = muxer!!.addTrack(format)
                    trackIndices[i] = newTrackIndex
                }
            }

            // ========== 剪辑阶段 ==========

            muxer!!.start()

            // 4. 跳到开始时间（关键帧）
            extractor!!.seekTo(startTimeMs * 1000, MediaExtractor.SEEK_TO_CLOSEST_SYNC)

            // 5. 循环处理数据
            val buffer = ByteBuffer.allocate(1024 * 1024)
            val bufferInfo = MediaCodec.BufferInfo()

            while (true) {
                // 读取一个样本
                val sampleSize = extractor!!.readSampleData(buffer, 0)

                // 没有更多数据就退出
                if (sampleSize < 0) {
                    break
                }

                val sampleTime = extractor!!.sampleTime

                // 超过结束时间就退出
                if (sampleTime > endTimeMs * 1000) {
                    break
                }

                // 只处理时间范围内的数据
                if (sampleTime >= startTimeMs * 1000) {
                    bufferInfo.offset = 0
                    bufferInfo.size = sampleSize
                    bufferInfo.presentationTimeUs = sampleTime - startTimeMs * 1000
                    bufferInfo.flags = extractor!!.sampleFlags

                    // 获取当前轨道的新索引
                    val currentTrack = extractor!!.sampleTrackIndex
                    val newTrackIndex = trackIndices[currentTrack]

                    if (newTrackIndex != null) {
                        muxer!!.writeSampleData(newTrackIndex, buffer, bufferInfo)
                    }
                }

                // 移动到下一个样本
                extractor!!.advance()
            }

            // ========== 完成阶段 ==========

            muxer!!.stop()

        } finally {
            release()
        }
    }

    /**
     * 释放资源
     */
    fun release() {
        try {
            muxer?.release()
            extractor?.release()
        } catch (e: Exception) {
            e.printStackTrace()
        }
    }
}
```

### 时间戳调整详解

```
🎯 为什么需要调整时间戳？

原始视频（0-10秒）：
帧1 @ 0ms
帧2 @ 33ms   →  偏移 = 3000ms
帧3 @ 66ms       新时间 = 66 - 3000 = -2934ms ❌ 负数不行！
...

如果我们从 3000ms 开始裁剪：
- 原时间戳：3000ms, 3033ms, 3066ms...
- 新时间戳：0ms, 33ms, 66ms... ✅

公式：
  新时间戳 = 原时间戳 - 开始时间
```

## 🎯 完整代码示例

### 示例 1：简单的视频裁剪工具类

```kotlin
package com.example.stability.video_edit

import android.media.MediaCodec
import android.media.MediaExtractor
import android.media.MediaFormat
import android.media.MediaMuxer
import android.util.Log
import java.io.File
import java.nio.ByteBuffer

/**
 * 视频裁剪工具类
 *
 * 使用方法：
 * val clipper = VideoTrimmer("/sdcard/input.mp4", "/sdcard/output.mp4")
 * clipper.trim(startTimeMs = 3000, endTimeMs = 7000)
 * clipper.release()
 */
class VideoTrimmer(
    private val inputPath: String,
    private val outputPath: String
) {
    private val TAG = "VideoTrimmer"

    private var extractor: MediaExtractor? = null
    private var muxer: MediaMuxer? = null

    /**
     * 裁剪视频
     *
     * @param startTimeMs 开始时间（毫秒）
     * @param endTimeMs 结束时间（毫秒）
     * @return 是否成功
     */
    fun trim(startTimeMs: Long, endTimeMs: Long): Boolean {
        Log.d(TAG, "开始裁剪: $startTimeMs ms - $endTimeMs ms")

        try {
            // 步骤 1：检查输入文件
            val inputFile = File(inputPath)
            if (!inputFile.exists()) {
                Log.e(TAG, "输入文件不存在: $inputPath")
                return false
            }

            // 步骤 2：检查输出目录
            val outputFile = File(outputPath)
            val parentDir = outputFile.parentFile
            if (parentDir != null && !parentDir.exists()) {
                parentDir.mkdirs()
            }

            // 步骤 3：设置数据源
            extractor = MediaExtractor()
            extractor!!.setDataSource(inputPath)

            // 步骤 4：创建输出文件
            muxer = MediaMuxer(outputPath, MediaMuxer.OutputFormat.MUXER_OUTPUT_MPEG_4)

            // 步骤 5：处理轨道
            val trackIndexMap = mutableMapOf<Int, Int>() // 原轨道 -> 新轨道

            for (i in 0 until extractor!!.trackCount) {
                val format = extractor!!.getTrackFormat(i)
                val mime = format.getString(MediaFormat.KEY_MIME) ?: continue

                // 只处理视频和音频轨道
                if (mime.startsWith("video/") || mime.startsWith("audio/")) {
                    Log.d(TAG, "找到轨道 $i: $mime")
                    extractor!!.selectTrack(i)
                    val newTrackIndex = muxer!!.addTrack(format)
                    trackIndexMap[i] = newTrackIndex
                }
            }

            // 步骤 6：开始混合
            muxer!!.start()

            // 步骤 7：跳到开始位置
            Log.d(TAG, "跳转到开始位置: ${startTimeMs * 1000} us")
            extractor!!.seekTo(startTimeMs * 1000, MediaExtractor.SEEK_TO_CLOSEST_SYNC)

            // 步骤 8：循环处理数据
            val buffer = ByteBuffer.allocate(1024 * 1024)
            val bufferInfo = MediaCodec.BufferInfo()
            var frameCount = 0
            var audioCount = 0

            while (true) {
                val sampleSize = extractor!!.readSampleData(buffer, 0)

                if (sampleSize < 0) {
                    Log.d(TAG, "数据读取完成")
                    break
                }

                val sampleTime = extractor!!.sampleTime
                val sampleTrack = extractor!!.sampleTrackIndex

                // 检查是否超过结束时间
                if (sampleTime > endTimeMs * 1000) {
                    Log.d(TAG, "已超过结束时间: $sampleTime > ${endTimeMs * 1000}")
                    break
                }

                // 计算新的时间戳（从0开始）
                val newTimeUs = sampleTime - startTimeMs * 1000

                if (newTimeUs >= 0) {
                    bufferInfo.offset = 0
                    bufferInfo.size = sampleSize
                    bufferInfo.presentationTimeUs = newTimeUs
                    bufferInfo.flags = extractor!!.sampleFlags

                    // 获取对应的输出轨道
                    val outputTrack = trackIndexMap[sampleTrack]
                    if (outputTrack != null) {
                        muxer!!.writeSampleData(outputTrack, buffer, bufferInfo)

                        // 统计
                        val mime = extractor!!.getTrackFormat(sampleTrack)
                            .getString(MediaFormat.KEY_MIME)
                        if (mime?.startsWith("video/") == true) {
                            frameCount++
                        } else if (mime?.startsWith("audio/") == true) {
                            audioCount++
                        }
                    }
                }

                // 移动到下一个样本
                extractor!!.advance()
            }

            Log.d(TAG, "裁剪完成: $frameCount 帧视频, $audioCount 帧音频")
            return true

        } catch (e: Exception) {
            Log.e(TAG, "裁剪失败", e)
            return false
        } finally {
            release()
        }
    }

    /**
     * 释放资源
     */
    fun release() {
        try {
            muxer?.stop()
            muxer?.release()
            extractor?.release()
        } catch (e: Exception) {
            Log.e(TAG, "释放资源失败", e)
        }
    }
}
```

fun splitVideo(inputPath: String, output1: String, output2: String, splitTimeMs: Long) {

    trimmer2.release()
}

## 🎯 常见问题解答

### Q1: 为什么裁剪后的视频时间不对？

**A**: 时间戳调整错误。

```kotlin
// ❌ 错误：没有调整时间戳
bufferInfo.presentationTimeUs = extractor.sampleTime

// ✅ 正确：减去开始时间
bufferInfo.presentationTimeUs = extractor.sampleTime - startTimeMs * 1000
```

### Q2: 为什么视频开头有黑屏或花屏？

**A**: 没有从关键帧开始。

```kotlin
// ❌ 错误：使用 SEEK_TO_PREVIOUS_SYNC 或 SEEK_TO_NEXT_SYNC
extractor.seekTo(startTimeMs * 1000, MediaExtractor.SEEK_TO_PREVIOUS_SYNC)

// ✅ 正确：使用 SEEK_TO_CLOSEST_SYNC
extractor.seekTo(startTimeMs * 1000, MediaExtractor.SEEK_TO_CLOSEST_SYNC)
```

### Q3: 为什么音频和视频不同步？

**A**: 视频和音频轨道需要分别调整时间戳。

```kotlin
// 视频时间戳
val newVideoTime = videoTime - startTimeMs * 1000

// 音频时间戳
val newAudioTime = audioTime - startTimeMs * 1000

// 两者必须分开处理
```

### Q4: 为什么输出文件打不开？

**A**: 可能的原因：
1. 没有正确添加所有轨道
2. Muxer 没有 start 就写入数据
3. 时间戳为负数

```kotlin
// ✅ 确保在 start() 之后才写入
muxer.start() // 先 start
muxer.writeSampleData(...) // 再写入
```

### Q5: 如何处理大视频？

**A**: 对于大视频，需要分批处理，避免内存溢出。

```kotlin
// 使用 streaming 方式，不要一次性加载整个视频
while (true) {
    val sampleSize = extractor.readSampleData(buffer, 0)
    if (sampleSize < 0) break

    // 处理当前样本
    processSample(buffer, bufferInfo)

    extractor.advance()
}
```

### Q6: 如何获取视频时长？

**A**: 使用 MediaMetadataRetriever。

```kotlin
fun getVideoDuration(path: String): Long {
    val retriever = MediaMetadataRetriever()
    retriever.setDataSource(path)

    val duration = retriever.extractMetadata(
        MediaMetadataRetriever.METADATA_KEY_DURATION
    )?.toLongOrNull() ?: 0L

    retriever.release()
    return duration
}
```
